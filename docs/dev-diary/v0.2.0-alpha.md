# Dev Diary: v0.2.0-alpha

**Date:** 2026-06-26  
**Author:** Shiv Singh  
**Release:** Kernel foundation

## What was added

- `panic()` - visible kernel panic instead of silent triple faults
- Heap (`kmalloc` / `kfree`) with a simple free-list
- Identity-mapped paging for the first 4 MB
- PIT timer at 100 Hz and `uptime` command
- IRQ-driven keyboard input
- Regression checklist and seven architecture decision records

## What I learned

- Splitting heap and paging into separate steps makes debugging much easier
- Page faults must route to `panic()` before chasing other bugs
- `sti; hlt` in the shell read loop saves CPU while waiting for IRQ keyboard data

## Challenges solved

- Enabled paging without breaking the kernel at 0x1000 (identity map)
- Wired IRQ 0 (PIT) and IRQ 1 (keyboard) through the existing IDT/PIC setup
- Heap self-test at boot catches allocator regressions early
- Fixed IRQ bring-up: ISR stubs must live in `.text` (not `.note.GNU-stack`); `isr_common` and `registers_t` now follow the OSDev stack layout; exception vectors 8 and 17 use the correct `ISR_NOERR` / `ISR_ERR` macros

## IRQ fix (2026-06-26)

`make run-debug` now reaches `BOOT 8` with `sti` enabled. PIT IRQ 0 ticks at 100 Hz (`uptime` increments). Keyboard uses IRQ 1 with `sti; hlt` in the read loop (no polling fallback).
